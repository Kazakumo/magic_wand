# 要件定義 — magic_wand（分散型AI学習オーケストレーター）

最終更新: 2026-03-27 (v2 — TUI・進捗管理・インジェスト最適化を追加)

---

## 1. システム概要

**magic_wand** は、ユーザーの学習活動（読んだ本・技術記事・書いたコード・学習ログ）を
ローカルで完結するAIパイプラインに取り込み、以下を実現するシステムである。

| 機能軸 | 説明 |
|--------|------|
| 自動収集 | 指定ディレクトリを監視し、新規・更新ファイルをリアルタイムでベクトル化 |
| 検索・生成 | RAGエンジンで「過去の自分の知識」を文脈として使いながらLLMが回答 |
| 推薦 | セマンティック類似度 × 時刻重みづけ × 習熟度で「次に学ぶべき内容」を提示 |
| 進捗管理 | 学習アイテムのステータス・習熟度をTUIから直接操作 |
| TUI | 美しいターミナルUIで全機能を統合操作 |

クラウド不要・ローカルファーストで動作する。

---

## 2. ユーザーストーリー

### US-01 学習ログのインジェスト
> エンジニアとして、毎日書いているメモや読んだ本のハイライトを指定フォルダに置くだけで、
> 自動的にAIが学習可能な形に変換してほしい。

**受け入れ条件**
- `.md` / `.txt` / `.go` / `.py` 等のテキストファイルを監視対象として登録できる
- ファイルの新規作成・更新を検知したら60秒以内にベクトル化が完了する
- CPU使用率が閾値（デフォルト70%）を超えたら自動的にワーカー数を下げ、負荷を抑制する
- ファイルのSHA256ハッシュを記録し、内容が変わっていない場合はEmbeddingをスキップする

### US-02 セマンティック検索
> 「Goのエラーハンドリング」と質問したら、自分がこれまで書いたコードや読んだ本の中から
> 関連する箇所を引っ張ってきて答えてほしい。

**受け入れ条件**
- 自然言語クエリを入力するとベクトル検索で上位k件（デフォルト10）を取得する
- コサイン類似度 + コンテキスト密度（キーワード密度・AST解析）でリランキングする
- LLMがリランキング後のコンテキストを使って回答を生成する（RAG）
- TUIのQueryビューで回答をストリーミング表示し、参照ソースをインラインで示す

### US-03 学習推薦
> 「最近Goのgoroutineにハマっている」という状況を伝えたら、
> 過去の学習内容から関連するものを引き出して「次に読むべきもの」を提示してほしい。

**受け入れ条件**
- 現在の悩み・キーワードを入力すると、過去コンテンツとのセマンティック距離を計算する
- 時刻重みづけ（exponential decay）を組み合わせてランキングを生成する
- **未習得（unread/reading）のアイテムを優先**して推薦する
- 「なぜこれを推薦するか」の根拠（参照元・類似度スコア・習熟度）を表示する

### US-04 コンテンツ管理
> 削除・リネームしたファイルはインデックスからも消えてほしい。

**受け入れ条件**
- ファイル削除検知でベクトルDBのエントリを削除する
- リネーム時はメタデータ（パス・ハッシュ）を更新する

### US-05 学習進捗管理
> TUIのLibraryビューで、各ファイルのステータス（未読・読中・習得済）と
> 習熟度（0〜100%）を一覧で確認・更新したい。

**受け入れ条件**
- LibraryビューにStatusとMasteryカラムを持つテーブルを表示する
- キー操作（`r`=reading, `m`=mastered, `u`=unread）でステータスを切り替えられる
- Masteryは「このファイルに関するアウトプット（メモ・コード）との類似度」から自動計算される
- フィルタ（All / Unread / Reading / Mastered）でリストを絞り込める

### US-06 統合TUIダッシュボード
> ターミナルを開いてコマンドを叩くだけで、美しいUIで全機能にアクセスしたい。

**受け入れ条件**
- `magic_wand` を引数なしで起動するとTUIが立ち上がる
- タブキーまたは数字キーで5つのビュー（Dashboard / Library / Query / Recommend / Ingest）を切り替えられる
- ボトムバーに常にシステム状態（Watcher ON/OFF・ワーカー数・CPU使用率・インデックス件数）を表示する
- `?` キーでヘルプオーバーレイを表示する

## 3. 機能要件

### 3.1 インジェスション・パイプライン

| # | 要件 |
|---|------|
| F-01 | fsnotifyによるファイル監視（再帰的なディレクトリ対応） |
| F-02 | テキストチャンキング（オーバーラップ付きスライディングウィンドウ） |
| F-03 | Workerプール（goroutine）による並列Embedding生成 |
| F-04 | CPU使用率に応じた動的ワーカー数制御 |
| F-05 | ベクトルDBへの書き込み（バッチアップサート） |
| F-06 | ファイル削除・移動のイベント処理 |
| F-07 | **SHA256ハッシュによる重複スキップ**（内容未変更ファイルはEmbeddingしない） |
| F-08 | イベントデバウンス（500ms）によるfsnotify多重発火の抑制 |

### 3.2 Embeddingレイヤー

| # | 要件 |
|---|------|
| F-09 | Ollamaのembeddingモデル（`nomic-embed-text`等）との連携 |
| F-10 | モデル設定の切り替え対応（config経由） |
| F-11 | リトライ・タイムアウト制御（`context.Context`） |

### 3.3 ベクトルDBレイヤー

| # | 要件 |
|---|------|
| F-12 | QdrantとのgRPC通信 |
| F-13 | コレクション自動作成・スキーマ管理 |
| F-14 | 拡張メタデータの保存（パス・タイムスタンプ・チャンクID・**ハッシュ・ステータス・習熟度**） |
| F-15 | 類似度検索（Cosine Similarity、上位k件） |
| F-16 | フィルタリング（ファイル種別・日付範囲・**status・mastery_level**） |
| F-17 | ステータス・習熟度のメタデータ更新（TUIからの操作を反映） |

### 3.4 RAGエンジン

| # | 要件 |
|---|------|
| F-18 | クエリのベクトル化 → 検索 → リランキングのパイプライン |
| F-19 | **多段階リランキング**: コサイン類似度 × キーワード密度 × AST重み |
| F-20 | LLMへのプロンプト構築（コンテキスト注入） |
| F-21 | OllamaのチャットAPIへのストリーミングリクエスト |
| F-22 | 回答とその根拠（ソース一覧・スコア）の返却 |

### 3.5 セマンティック推薦エンジン

| # | 要件 |
|---|------|
| F-23 | 時刻重みづけスコアリング（exponential decay） |
| F-24 | 入力クエリと過去コンテンツの類似度計算 |
| F-25 | **習熟度フィルタリング**（未習得を優先、習得済を降順に配置） |
| F-26 | 推薦理由の自然言語生成 |
| F-27 | 推薦結果のランキング表示 |

### 3.6 TUI（ターミナルUI）

| # | 要件 |
|---|------|
| F-28 | **Dashboardビュー**: 統計カード・最近の活動・システムヘルス |
| F-29 | **Libraryビュー**: 学習アイテムテーブル・ステータス操作・フィルタ |
| F-30 | **Queryビュー**: チャットUI・ストリーミング回答・ソースパネル |
| F-31 | **Recommendビュー**: 推薦ランキング・スコアバッジ・理由表示 |
| F-32 | **Ingestビュー**: リアルタイム進捗・ワーカー可視化・CPU計 |
| F-33 | グローバルステータスバー（常時表示） |
| F-34 | ヘルプオーバーレイ（`?`キー） |
| F-35 | カラーテーマ（magic_wand専用パレット） |

### 3.7 非インタラクティブCLI（スクリプト用）

| # | 要件 |
|---|------|
| F-36 | `magic_wand ingest <path>` — 手動インジェスト |
| F-37 | `magic_wand query "<text>"` — 非インタラクティブ検索 |
| F-38 | `magic_wand watch` — バックグラウンドウォッチャー起動 |

## 4. 非機能要件

| 分類 | 要件 |
|------|------|
| パフォーマンス | 1ファイル（〜10KB）のベクトル化を10秒以内に完了 |
| リソース | CPU使用率閾値設定でピーク時の負荷を抑制 |
| 無駄排除 | SHA256ハッシュで未変更ファイルのEmbeddingをスキップ |
| 可観測性 | `slog`による構造化ログ（JSON出力対応） |
| ポータビリティ | Docker Composeでローカル依存サービスを一括起動 |
| テスタビリティ | 各コンポーネントはinterfaceで抽象化しモック差し替え可能 |
| セキュリティ | ローカルのみで動作・外部ネットワーク不要 |
| UX | TUIはリサイズ対応・60fps相当のレンダリング |
